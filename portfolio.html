Retail Store Interface with Node and MySQL

The following CLI application let's users browse retail inventory and select a quantity of units to purchase.
The app deploys in the Node.js environment. Output is viewed on the command prompt/ command line/ PowerShell etc.,

Running the Application
It is not necessary to run the application with argv.params. The application uses npm's inquirer to walk the user through the functionality. The inquirer commands prompt the following:

  • id of the product of interest

  • number of units to purchase


    • Upon receiving user input on product id and the number of units, the application confirms the product name with the user and confirms with the database that there is sufficient inventory. If there is insufficient inventory, the user is informed and prompted on selecting a different product, if desired. After every product submission, the user is prompted to determine whether or not they'd like to submit an additional order. Learn more with the screenshots provided below!

    The Friend Finder Application

    Users are prompted to take a short 10 question personality quiz. Data, via a POST method, is sent to the server for processing. Processing consists of comparing user scores, stored in an array of objects generated by user participation, to one another to find users with the most compatible personalities, measured according to their survey score.

    The application is hosted here but can also be run from a local server (8080). Running a local server requires the following code in the terminal:

    ``` nodemon server.js ```

    Burgers and the Model-View-Controller

    The burger app is about more than just burgers. It showcases GET, POST, and PUT requests using AJAX calls to interact with a database using the Model-View-Controller(MVC) architecture. The application uses callback functions in both the ORM, the models and the controllers to communicate request and response data. In this case, the res and req data is: 1) the name of the burgers and 2)a boolean variable representing devoured or eaten as true or false. This application also includes a custom ORM that as aforementioned communicates seamlessly with the model, the view(s), and the controller. Stay tuned for the next iteration of the application which will use the popular Sequelize as the ORM.

    The application is hosted here but can also be run from a local server (8080). Running a local server requires the following code in the terminal:

    ``` nodemon server.js ```
